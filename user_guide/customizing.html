自定义主题#

除了在 配置 中详细介绍的配置选项外，还可以自定义主题的 HTML 布局和 CSS 样式。

危险

This theme is still under active development, and we make no promises about the stability of any specific HTML structure, CSS variables, etc. Make these customizations at your own risk, and pin versions if you’re worried about breaking changes!

自定义主题样式#

你可以通过创建定制样式表来定制主题的 CSS, Sphinx 使用它来构建你的网站。这个样式表中的任何规则都将覆盖默认的主题规则。

按照以下步骤添加自定义样式表：

  1. 在 _static/css/custom.css 中 创建 CSS 样式表，并添加你希望的 CSS 规则。

  2. 将样式表附加到 Sphinx 构建中。将以下内容添加到 conf.py 中

    html_static_path = ['_static']

html_css_files = [
    'css/custom.css',
]

当您构建文档时，这个样式表现在应该被激活。

Manage themes#

危险

Theming is still a beta feature so the variables related to the theme switch are likely to change in the future. No backward compatibily is guaranteed when customization is done.

Pydata sphinx theme embed 3 different theming mode:

  • auto: the documentation theme will follow the one provided by your computer

  • dark: the documentation is displayed with the dark theme

  • light: the documentation is displayed with the light theme

In order to customize the display of any of the theme element you need to encaspulate your modifications in the approriate css rules:

/* anything related to the light theme */
html[data-theme="light"] {

    /* whatever you want to change */
    background: white;
}

/* anything related to the dark theme */
html[data-theme="dark"] {

    /* whatever you want to change */
    background: black;
}

A complete list of the used colors for this theme can be found in the pydata default css colors file.

CSS 主题变量#

这个主题定义了几个 CSS 变量，可以用来快速控制整个文档的行为。

这些是基于基本的 Bootstrap CSS 变量 扩展了一些主题特定的变量。所有变量和每个默认值的概述都定义在 pydata 默认 CSS 变量文件。

要更改变量，请遵循以下步骤：

  1. 添加自定义的 CSS 样式表 这就是我们配置变量的地方。

  2. 在 :root 部分下面，添加你想要更新的变量。例如，要更新基本字体大小，你可以把它添加到 custom.css 中：

    :root {
    --pst-font-size-base: 17px;
}

重要

Note that these are CSS variables and not SASS variables. The theme is defined with CSS variables, not SASS variables! Refer to the previous section if you desire a different behavior between the light and dark theme.

关于你可以覆盖的主题变量的完整列表，请参阅 主题变量默认 CSS 文件：

/* Provided by Sphinx's 'basic' theme, and included in the final set of assets */
@import "../basic.css";

  /*****************************************************************************
  * Theme config
  **/
  --pst-header-height: 60px;

  /*****************************************************************************
  * Font size
  **/
  --pst-font-size-base: 15px; /* base font size - applied at body / html level */

  /* heading font sizes */
  --pst-font-size-h1: 36px;
  --pst-font-size-h2: 32px;
  --pst-font-size-h3: 26px;
  --pst-font-size-h4: 21px;
  --pst-font-size-h5: 18px;
  --pst-font-size-h6: 16px;

  /* smaller then heading font sizes*/
  --pst-font-size-milli: 12px;

  --pst-sidebar-font-size: 0.9em;
  --pst-sidebar-caption-font-size: 0.9em;

  /*****************************************************************************
  * Font family
  **/
  /* These are adapted from https://systemfontstack.com/ */
  --pst-font-family-base-system: -apple-system, BlinkMacSystemFont, Segoe UI,
    "Helvetica Neue", Arial, sans-serif, Apple Color Emoji, Segoe UI Emoji,
    Segoe UI Symbol;
  --pst-font-family-monospace-system: "SFMono-Regular", Menlo, Consolas, Monaco,
    Liberation Mono, Lucida Console, monospace;

  --pst-font-family-base: var(--pst-font-family-base-system);
  --pst-font-family-heading: var(--pst-font-family-base);
  --pst-font-family-monospace: var(--pst-font-family-monospace-system);

  /*****************************************************************************

  /* font awesome icons*/
  --pst-icon-check-circle: "\f058";

  --pst-icon-admonition-note: var(--pst-icon-info-circle);

  --pst-icon-admonition-danger: var(--pst-icon-exclamation-triangle);

替换/删除字体#

主题包括 FontAwesome 5 Free 图标字体（.fa, .far, .fas 样式，用于 icon links 和警告)。这是唯一的 vendored 字体，否则主题默认依赖于正常的正文文本和标题可用的系统字体。

注意

以前包括的字体，如 Lato 已经被删除，而选择读者计算机最常见的默认系统字体。这比自定义字体提供了更好的性能和更好的脚本/符号覆盖率，在大多数情况下都是推荐的。

默认的正文和标题字体可以更改如下：

  • 使用 自定义主题样式，你可以指定正文、标题和单空格文本使用的字体。例如，以下内容可以添加到自定义 css 文件中：

    :root {
    --pst-font-family-base: Verdana, var(--pst-font-family-base-system);
    --pst-font-family-heading: Cambria, Georgia, Times, var(--pst-font-family-base-system);
    --pst-font-family-monospace: Courier, var(--pst-font-family-monospace-system);
}

    -system 变量可用于回退到默认字体。

  • 如果你在上面一节中指定的字体在默认情况下通常是不可用的，你还需要确保字体被加载。例如，你可以在 Sphinx 网站的 _static 目录下下载并提供字体，然后更新基本模板来加载字体资源：

    • 在你的 conf.py 中配置 template_path

    • 创建自定义的 layout.html Jinja2 模板，它会重载 fonts 块（例如加载包含在 _static/vendor 目录中的 Lato 字体）：

      {% extends "pydata_sphinx_theme/layout.html" %}

{% block fonts %}
  <!-- add `style` or `link` tags with your CSS `@font-face` declarations here -->
  <!-- ... and optionally preload the `woff2` for snappier page loads -->
  <link rel="stylesheet" href="{{ pathto('_static/vendor/lato_latin-ext/1.44.1/index.css', 1) }}">

{% endblock %}

      为了减少 Flash of Unstyled Content，你可能希望探索各种选项的 预加载内容，特别是二进制字体文件。这可以确保在 CSS 被解析之前加载文件，但应该小心使用。
