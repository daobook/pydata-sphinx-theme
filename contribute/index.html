贡献#

这些页面包含关于如何启动和运行这个主题的开发版本的信息，以及如何为项目做出贡献。

贡献变更的工作流#

我们遵循 典型的 GitHub 工作流程：

  • 创建这个 repo 的个人分叉

  • 创建 branch

  • 打开 pull request

  • 修正了各种测试和检查的结果

  • 完成代码评审

对于每个 pull request，演示站点被构建和部署，以使它更容易审查 PR 中的变化。要访问这个，点击 “ReadTheDocs” 预览在 CI/CD 作业。

文档的位置和结构#

这个主题的文档在 docs/ 文件夹中。它的结构是 Sphinx 文档站点。内容是在 reStructuredText 和 MyST Markdown 的组合编写的。

CSS/JS assets 的位置和结构#

这个主题的 CSS 和 JS 是用 webpack 为浏览器 src/pydata_sphinx_theme/assets/* 构建的。主要的切入点是：

  • CSS: src/pydata_sphinx_theme/assets/styles/index.scss

    • 主题 assets 的 main 部分

    • 使用 Sass 定制 Bootstrap

  • JS: src/pydata_sphinx_theme/assets/scripts/index.js

    • 提供附加引导功能，以及一些自定义导航行为

  • webpack: webpack.config.js

    • 捕获了将 src/pydata_sphinx_theme/assets/* 中的 JS 和 CSS 源文件转换为 src/theme/pydata_sphinx_theme/static/ 中的生产 assets 的技术
