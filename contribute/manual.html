安装手册开发环境#

如果你不喜欢使用像 nox 这样的自动化工具，或者想要对安装的软件包的特定版本有更多的控制，你也可以手动在本地设置一个开发环境。

要做到这一点，请遵循本页上的说明。

创造新的开发环境#

这是可选的，但最好从新的开发环境开始，这样您就可以隔离用于这个存储库的包。

要做到这一点，请使用 conda、mamba 或 virtualenv 等工具。

克隆仓库到本地#

首先从 pydata 组织克隆这个存储库，或者从你已经创建的 fork 克隆：

$ git clone https://github.com/pydata/pydata-sphinx-theme
$ cd pydata-sphinx-theme

安装 sphinx-theme-builder#

使用 sphinx-theme-builder 在本地安装 nodejs，并编译主题所需的所有 CSS 和JS 资产。像这样安装它（注意 cli 选项，这样我们就可以从命令行运行它）：

$ pip install sphinx-theme-builder[cli]

在本地安装主题#

接下来，在本地安装这个主题，这样我们就有必要的依赖来构建文档和测试套件：

$ pip install -e .[dev]

请注意 sphinx-theme-builder 将自动安装 nodejs 的本地副本来构建主题的 assets。这将被放置在 .nodeenv 文件夹中。

构建文档#

手动构建文档，运行如下命令：

$ sphinx-build docs docs/_build/html

编译 web 资源(JS/CSS)#

编译主题的 javascript 和 CSS 资产，运行以下命令：

$ stb compile

这将编译 src/pydata_sphinx_theme/assets 文件夹中的所有内容，并将它们放在主题文件夹结构中的适当位置。

启动实时服务器来构建和服务您的文档#

要手动打开服务器查看文档中的更改，构建它们，并在浏览器中本地显示它们，运行以下命令：

$ stb serve docs --open-browser

运行测试#

若要手动运行此主题的测试，请先在本地设置环境，然后运行：

$ pytest
